CAPÍTULO 1

Arquitetura de Classes

O coração do seu RPG: 8 classes únicas, cada uma com 5 habilidades desbloqueáveis por nível (Level Cap 5).

Como funciona a progressão

Cada classe começa no Nível 1 com sua primeira habilidade. A cada novo nível, uma habilidade adicional é desbloqueada, até chegar ao Nível 5 com todas as 5 habilidades disponíveis. O Level Cap inicial é 5, mas você pode expandir depois.

// Habilidade de buff (Postura Defensiva)

CAPÍTULO 2

Engine de Combate

Sistema de turnos, Stagger, durabilidade de armas e a IA dos inimigos.

Sistema de Turnos

O combate é baseado em turnos com 4 ações principais:

  • [1] Atacar — Ataque básico com dano calculado dos atributos + arma
  • [2] Habilidades — Sub-menu com as skills disponíveis da sua classe
  • [3] Itens — Usar consumíveis do inventário
  • [4] Fugir — Chance de escapar baseada na agilidade

Barra de Stagger (Atordoamento)

Inimigos possuem uma barra de resistência (Stagger). Cada ataque reduz essa barra. Quando chega a zero:

  • O inimigo pula o turno dele
  • O jogador causa dano crítico garantido
  • A barra é resetada para o combate continuar
Stagger do Inimigo75%

Durabilidade de Armas

Cada ataque reduz a durabilidade da arma. Se chegar a 0, o dano cai 90% até ser reparada no Ferreiro. Os estados são:

Durabilidade100%
  • 100% — Normal
  • 50% — Desgastada
  • 25% — Danificada
  • 0% — Quebrada (dano -90%)
✨
Curiosidade: No código original da IA, a função calcular_ataque_total() leva em conta: dano base da arma, força do personagem, bônus de equipamento, nível e refino. Tudo vira uma fórmula única!
CAPÍTULO 3

Sistema Elemental

Fogo queima Gelo, Gelo congela Terra, Terra sepulta Fogo. Luz e Trevas em conflito eterno.

Círculo de Fraquezas

O sistema possui 6 elementos com uma cadeia de fraquezas dupla: acertar uma fraqueza causa DANO DOBRADO (2x), enquanto acertar uma resistência causa METADE DO DANO (0.5x).

⚔ Código em Portugol — Verificação Elemental

Portugol
funcao verificar_fracao_elemental(inteiro atacante_elem, inteiro defensor_elem)
    real multiplicador = 1.0
    
    // Fraquezas (DANO DOBRADO)
    se (atacante_elem == ELEM_FOGO e defensor_elem == ELEM_GELO)
        multiplicador = 2.0
    
    // Resistências (DANO REDUZIDO)
    se (defensor_elem == ELEM_FOGO e atacante_elem == ELEM_GELO)
        multiplicador = 0.5
    
    retorne multiplicador
fimfuncao

💻 Em Lua para Roblox

Lua (Roblox)
local ELEMENTOS = {
    FOGO = "Fogo",   GELO = "Gelo",
    TERRA = "Terra",  LUZ = "Luz",
    TREVAS = "Trevas", RAIO = "Raio"
}

local fraquezas = {
    [ELEMENTOS.FOGO]   = ELEMENTOS.GELO,
    [ELEMENTOS.GELO]   = ELEMENTOS.TERRA,
    [ELEMENTOS.TERRA]  = ELEMENTOS.FOGO,
    [ELEMENTOS.LUZ]    = ELEMENTOS.TREVAS,
    [ELEMENTOS.TREVAS] = ELEMENTOS.LUZ
}

function calcularMultiplicadorElemental(atkElem, defElem)
    if fraquezas[atkElem] == defElem then
        return 2.0  -- Dano dobrado!
    elseif fraquezas[defElem] == atkElem then
        return 0.5  -- Resistência
    end
    return 1.0
end
💡
Dica de Balanceamento: No Roblox, você pode usar Attribute ou Configuration objects para armazenar os elementos e fraquezas de cada inimigo, facilitando a criação de novos monstros sem editar código.
CAPÍTULO 4

Buffs, Debuffs e Efeitos

Um sistema profissional de efeitos temporários baseado em duração por turnos.

Arquitetura de Efeitos

Cada efeito possui: nome, duração (turnos), intensidade, tipo, origem, acumulável e prioridade. O sistema processa efeitos no início e fim de cada turno automaticamente.

Início do Turno

Aplicar dano contínuo (veneno, queimadura), aplicar regeneração, verificar cooldowns, atualizar duração

Durante o Turno

Ações do jogador/inimigo consideram buffs ativos (ex: Escudo absorve dano)

Fim do Turno

Reduzir duração dos efeitos, remover efeitos expirados, processar efeitos finais

Efeito Expirado

Quando duração ≤ 0: remove buff/debuff e exibe mensagem "[Efeito Encerrado] Fúria terminou."

CAPÍTULO 5

Sobrevivência e Mundo

Ciclo dia/noite, fome, ambientes e como eles afetam a jogabilidade.

☀ Ciclo Dia/Noite

A cada 5 ações do jogador, o tempo muda. Isso afeta diretamente a dificuldade:

  • ☀ Dia: Inimigos normais, preços padrão
  • 🌙 Noite: Inimigos +30% de dano, mas concedem +50% de XP. Itens de cura 25% mais caros, itens de Trevas 20% mais baratos
  • 👹 Mercador Negro: Aparece apenas à noite com itens raros e perigosos

🍔 Sistema de Fome

Variável de 0 a 100 que diminui com o tempo. Se chegar a 0, o jogador fica Exausto, perdendo 50% de todos os atributos. Coma para sobreviver!

Fome100%

Alimentos: Pão Velho (+15 Fome, 10g) | Ração de Viagem (+60 Fome, 35g)

🏞 Ambientes

Habilidades especiais podem mudar o ambiente da batalha, aplicando buffs e debuffs:

  • Nevasca (Mago Nv 3): Ambiente Gelo — reduz precisão do inimigo em 30%
  • Bomba de Fumaça (Ladino Nv 3): Ambiente Escuridão — jogador não pode ser atingido no próximo turno
  • Crescimento Exuberante (Druida Nv 3): Ambiente Floresta — regenera 10% de vida por turno

CAPÍTULO 8

Balanceamento de Jogo

A arte de equilibrar números para que o jogo seja desafiante mas justo.

Fórmulas Principais

No código da IA, todas as fórmulas de balanceamento estão centralizadas em funções específicas. Veja as principais:

AtributoFórmula
HP Máximo100 + (Vitalidade × 15) + (Bônus Equip × 15) + (Nível × 12)
Mana Máxima50 + (Inteligência × 8) + (Bônus Equip × 8) + (Nível × 10)
Ataque TotalDano Arma + (Força × 1.8) + Bônus Equip + (Nível × 8) + Bônus Refino (%)
Defesa TotalDefesa Base + Armadura + Bônus Equip + (Nível × 7)
Redução de DanoDefesa × 0.35 (limitado a 80%)

Peso Máximo30 + (Força × 1.5)

⚔ Código em Portugol — Cálculo de Ataque

Portugol
funcao calcular_ataque_total()
    inteiro base = g_arma_dano + (g_forca * 1.8) + (g_equip_forca * 1.8)
    inteiro bonus_nivel = g_nivel * 8
    inteiro refino_bonus = 0
    
    se (g_arma_nivel_refino == 1) refino_bonus = 10
    senao se (g_arma_nivel_refino == 2) refino_bonus = 25
    senao se (g_arma_nivel_refino == 3) refino_bonus = 40
    senao se (g_arma_nivel_refino == 4) refino_bonus = 60
    senao se (g_arma_nivel_refino == 5) refino_bonus = 100
    
    inteiro total = base + bonus_nivel + (base * refino_bonus / 100)
    retorne total

Para evitar números absurdos, a IA implementou Hard Caps (limites máximos):

Chance Crítica80%Sempre há chance de não crítico
Esquiva75%Inimigos sempre podem acertar

⚠
Atenção: Hard caps são essenciais em qualquer RPG. Sem eles, um jogador pode acumular bônus e se tornar invencível, quebrando a diversão. No Roblox, defina esses limites em um ModuleScript de configuração global.

1. Progressão Exponencial de XP

A fórmula xp = 100 * 1.45^nível faz com que subir de nível fique cada vez mais difícil. No nível 1 você precisa de 145 XP, no nível 5 já são ~430 XP. Isso mantém o jogador engajado sem chegar ao topo rápido demais.

2. Inflação Noturna

Durante a noite, itens de cura custam 25% a mais. Isso força o jogador a planejar: "Será que eu compro poções agora de dia, ou arrisco e espero?" Pequenas decisões como essa criam jogabilidade memorável.

3. Stagger como "Soft Enrage"

A barra de Stagger impede que inimigos sejam永久mente atordoados. Cada vez que o Stagger zera, o inimigo volta mais forte. Isso evita a tática chata de "stunlock infinito".

4. Custo vs. Recompensa

Habilidades como Sacrifício (Necromante) consomem vida do jogador para causar dano massivo. Isso cria momentos de tensão — "Será que vale a pena arriscar?" Num RPG do Roblox, essas habilidades geram os momentos mais épicos.

5. Multiplicadores e Sinergia

Fúria (+ATK) combinado com Vampirismo (cura baseada no dano) é uma sinergia poderosa. Sempre teste combinações de buffs. Se uma combinação for forte demais, ajuste os multiplicadores ou adicione um conflito (ex: Fúria reduz a cura recebida).

💻 Exemplo: Sistema de Combate em Lua

Lua (Roblox)
-- Módulo de Combate
local Combate = {}
Combate.__index = Combate

function Combate.novo(jogador, inimigo)
    local self = setmetatable({}, Combate)
    self.jogador = jogador
    self.inimigo = inimigo
    self.turno = 1
    self.stagger = inimigo:GetAttribute("StaggerMax") or 100
    self.staggerAtual = self.stagger
    return self
end

function Combate:aplicarDano(alvo, dano, elemento)
    local multiplicador = 1.0
    local defesa = alvo:GetAttribute("Defesa") or 0
    
    -- Verifica fraqueza elemental
    local elemAlvo = alvo:GetAttribute("Elemento")
    if elemento and elemAlvo then
        multiplicador = calcularMultiplicadorElemental(elemento, elemAlvo)
    end
    
    -- Redução de dano pela defesa (cap 80%)
    local reducao = math.min(defesa * 0.35, 80)
    local danoFinal = math.max(1, dano * multiplicador * (1 - reducao/100))
    
    -- Reduz stagger
    self.staggerAtual = math.max(0, self.staggerAtual - danoFinal * 0.2)
    if self.staggerAtual <= 0 then
        -- Atordoado!
        alvo:SetAttribute("Atordoado", true)
        self.staggerAtual = self.stagger
    end
    
    return danoFinal
end

return Combate
